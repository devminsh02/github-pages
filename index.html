Review Frame

리뷰를 쌓기 좋은 기본 형식

매번 같은 틀로 적어두면 논문끼리 비교하기 쉬워지고, 나중에 프로젝트 아이디어로 다시 꺼내보기도 편해집니다.

  • 1. Problem 이 논문이 무엇을 해결하려는가

    동기, 기존 방식의 병목, 연구 질문을 한 단락으로 정리합니다.

  • 2. Method 핵심 아이디어와 실험 설계

    새로운 요소가 무엇인지, 어떤 설정에서 강점을 보였는지 기록합니다.

  • 3. Critique 한계와 의문점

    재현성, 데이터 편향, 실제 적용 가능성 같은 질문을 남깁니다.

  • 4. Follow-up 내 작업과 연결되는 지점

    직접 구현해볼 실험이나 관련 프로젝트 링크를 붙여두기 좋습니다.
